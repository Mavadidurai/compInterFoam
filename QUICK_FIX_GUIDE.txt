┌───────────────────────────────────────────────────────────────────────────┐
│ ❓ WHY IT FAILS                                                           │
└───────────────────────────────────────────────────────────────────────────┘

In controlDict, these are DIMENSIONED values:
    Tvap    [0 0 0 1 0 0 0] 3560;      ← dimensionedScalar object
            └──────────────┘
            dimensions

OpenFOAM stores them as "dimensionedScalar", not plain "scalar".
lookupOrDefault<scalar>() can't find them → returns default (0.0)

┌───────────────────────────────────────────────────────────────────────────┐
│ ✅ THE FIX                                                                │
└───────────────────────────────────────────────────────────────────────────┘

CORRECTED CODE:

if (phaseChangeDict.found("Tvap"))
{
    dimensionedScalar Tvap_dim(phaseChangeDict.lookup("Tvap"));
    input_Tvap = Tvap_dim.value();  // Extract the numeric value
}

Apply this pattern to ALL dimensioned parameters:
✓ Tvap, Tsol, hf, gasConstant, p_ref, maxSource (phaseChangeCoeffs)
✓ Cl, De (twoTemperatureProperties)

Keep lookupOrDefault<scalar>() for dimensionless parameters:
✓ evaporationCoeff, relaxationTime, alphaMin, etc.
